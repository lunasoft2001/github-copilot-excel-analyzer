# Contributing

Thanks for your interest in contributing to vbaExcel.

## How to contribute

1. Fork the repo
2. Create a branch: `git checkout -b feat/my-change`
3. Make your changes
4. Run a quick sanity check (export/import on a sample XLSM)
5. Commit with a clear message
6. Open a pull request

## Guidelines

- Keep the bundle clean: only include installable files in `skill-bundle/`.
- Avoid adding large docs or samples to the bundle.
- Prefer small, focused PRs.
- Use ASCII in files unless the file already contains non-ASCII.

## Reporting issues

Open an issue with:
- Steps to reproduce
- Expected vs actual behavior
- OS and Excel version
- Any error output
